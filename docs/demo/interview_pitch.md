# Interview Pitch

## 30-second version

I built a Python-based business automation agent that cleans spreadsheet data, validates customer/contact records, detects duplicates and missing fields, prepares CRM and QuickBooks-ready outputs, and adds safety controls such as human approval, role-based permissions, audit logs, rollback, and backups. The app has a Streamlit dashboard, SQLite storage, scheduled jobs, email notifications, Docker deployment, automated tests, CI/CD, and code-quality checks.

## 60-second version

This project solves a common business problem: companies often keep customer, sales, and invoice data in messy spreadsheets before sending it into CRM or accounting systems. My app reads CSV, Excel, or spreadsheet-style data, cleans and validates it, separates rejected rows and duplicates, generates reports, and prepares safe CRM/QuickBooks payloads. I designed the architecture to be modular, so connectors like HubSpot and QuickBooks can be added safely. I also added a dashboard, login system, roles, approval history, audit logs, scheduled jobs, email notifications, SQLite database storage, backup/restore, file versioning, Docker deployment, pytest tests, GitHub Actions, and Ruff/Black quality checks.

## Resume bullets

- Built a Python/Streamlit business automation agent for spreadsheet cleaning, CRM-ready exports, and QuickBooks sync planning.
- Implemented validation, duplicate detection, rejected-row handling, audit logs, approval history, role-based permissions, and safe dry-run API workflows.
- Added SQLite persistence, scheduled jobs, email notifications, backup/restore, file versioning, Docker deployment, pytest tests, CI/CD, and code-quality checks.
